# 偃武和平鸽助手

> 一个为偃武SLG手游做的自动化脚本引擎，现在我不太有时间维护了，希望有兴趣的朋友可以接着搞下去 —— 鸽|慕逸

## 写在前面

这个项目最开始是我们盟里自己用的辅助工具，主要实现自动集结、采集、日常任务、开荒这些等。技术上用了 **OCR + OpenCV 模板匹配 + 百分比坐标** 三重识别，理论上不同分辨率都能适配。目前引擎主体基本跑通了，但我个人时间实在跟不上，**最卡脖子的地方其实是界面识别一直不太准**，导致很多功能虽然代码写完了，但实际效果总差那么点意思。如果你对图像识别、自动化控制有研究，或者刚好也在玩这个游戏，欢迎来继续整活。

## 技术栈

- **语言**：java、Kotlin  
- **UI**：Jetpack Compose + Material3  
- **OCR**：Google ML Kit 中文文字识别（bundled）  
- **图像匹配**：OpenCV 模板匹配  
- **截图**：MediaProjection（VirtualDisplay）  
- **触控模拟**：Instrumentation + Accessibility 双模式  
- **架构**：多进程（main / enginfloat / daemon / remote）  
- **Native**：JNI（C++）

## 当前主要阻塞点

**识别模块不够稳定**  
目前用的 OCR 和模板匹配在某些界面下识别率不高，坐标偏移也偶尔出问题，导致很多后续逻辑跑不通。如果你对图像识别、ML Kit、OpenCV 或者更轻量的特征匹配有经验，这块是最大的突破口。

另外，`assets/templates/` 目录下的模板图片几乎还是空的（目前主要靠 OCR 硬撑），需要有人根据实际游戏界面补充模板资源；部分百分比坐标也需要在真机上慢慢校准。

Native JNI 层有些功能还是桩实现，如果涉及到注入或底层模拟，可能需要补全。
